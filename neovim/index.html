2026年我的Neovim配置

从大学开始，我一直在折腾Neovim编辑器，它开源、快速、高度客制化，符合我对编辑器的一切要求。虽然代码没写多少，两三年也积累了不少设置，光是包管理器就从vim-plug换到packer.nvim再到lazy.nvim。这次趁着春节假期，简单整理一下配置。

其实我的编程需求很简单：

  1. 用Rust写一些小CLI；
  2. 用R做数据分析和画图；
  3. 用Python写算法和Manim库做动画;
  4. 用Latex套模板写文章;
  5. 用lua写Neovim配置

所以配置不需要太复杂。

包管理器

我没有什么特殊需求，现在WSL中的Neovim的加载速度也还可以，所以目前没有理由再切换到Lazy.nvim，至于on-demand loading，作者是这么说的

A properly implemented Vim plugin should already load lazily without any help from a plugin manager (:help autoload). So there are few cases where these options actually make much sense. Making a plugin load faster is the responsibility of the plugin developer, not the user. If you find a plugin that takes too long to load, consider opening an issue on the plugin’s issue tracker.

所以让插件快速加载是插件作者的事，而不是包管理器应该干的事（虽然vim-plug也可以实现）。但是对于我没什么影响，毕竟我的启动速度已经够快，而且vim-plug也足够好用。

最主要的是telescope、lsp一些插件、vimtex，R.nvim，剩下的都是一些功能性的插件，比如leap用来快速移动，Comment用来注释代码，autopairs用来生成成对括号等。

至于Coc，它固然好用，安装方便，server直接CocInstall就能安装，但是有段时间r-languaguerserver一直报错（可能不是Coc的问题），所以我换成了Neovim内置的Lsp，感觉稳定一些。lsp我也没有下载多少，主要还是r-languageserver、rust_analyzer、basepyright和emmylua_ls

至于别的，基本是安装插件的基本配置，不再赘述。这样一套配置下来，基本可以满足我的编程、写作需求，所以我近期不打算再调整配置，把精力放在好好写程序和研究问题上。
